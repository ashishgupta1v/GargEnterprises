What's NOT Included (and Why)

Costs deliberately excluded from Phase 1 — and when they make sense

🌐
Multiple Domains DEFERRED
gargkitchens.com and renvox.co.in are not needed until Phase 2 when those brands are being actively marketed digitally. One domain (gargenterprises.co.in) is enough for Phase 1 — the app, API, and admin panel all run under it.
🍎
Apple Developer Program DEFERRED
₹8,999/yr for iOS access. Phase 1 launches on Android only — all Garg Enterprises staff testing happens on Android. iOS rollout makes sense in Phase 2 when the app is proven.
💬
WhatsApp Business API DEFERRED
₹1,200/mo for WhatsApp automation. Nice to have, but Phase 1 works fine with push notifications (free Firebase) + SMS for critical alerts. Add WhatsApp in Phase 2.
📧
Google Workspace Email OPTIONAL
₹900/mo for professional email (pankaj@gargenterprises.co.in). Genuinely useful for the business image, but not required for the app to function. Pankaj's call — it's a business decision, not a technical one.
⚖️
Load Balancer PHASE 2
₹240/mo for high-availability setup. Completely unnecessary for a single-store app with 10 users. Add when Phase 2 brings multi-location or when concurrent users exceed 50.
📊
Tally / GST Integration APIs PHASE 3
₹500–2,000/mo for Tally sync or GST filing APIs. Phase 1 is inventory-only — no billing, no GST, no accounting. These are Phase 3 features explicitly.

Pre-Development Timeline

What gets done day-by-day before a single line of code is written

Day 1–2 · Accounts & Kickoff
Pankaj creates Google account + DigitalOcean account + releases 50% development payment. Digital Builders registers domain, sets up Cloudflare DNS, creates Firebase project. Dev environment scaffolded on DO droplet.
Day 3–5 · Server + Database Live
DigitalOcean droplet provisioned. Laravel 12 project deployed. PostgreSQL database schema created. Redis + MeiliSearch configured on same server. CI/CD pipeline via GitHub Actions set up. API health check endpoint goes green.
Day 5–8 · Auth + Sample Data
Pankaj sends staff list + product data. OTP login system live. User roles configured. Sample SKU data loaded into database. Pankaj gets first internal test build on his Android phone.
Day 8–20 · Module 1 — Catalog
SKU hierarchy, full-text search, bulk Excel import, QR label generation, product detail screens. First alpha build delivered. Feedback collected from Pankaj. DLT SMS registration approved (started Day 1).
Day 20–34 · Module 2 — Inventory
Movement recording (all 6 types), maker-checker approval workflow, stock alerts engine (6 AM daily), reports. All 4 user roles tested together on real devices. UAT with Godown staff begins.
Day 34–42 · QA + Training + Go-Live
Performance benchmarking on 26K SKU search. Staff training. Historical stock migration from Excel. Google Play Store submission (if Play Console ready). Final delivery · 50% payment released. App is live.
