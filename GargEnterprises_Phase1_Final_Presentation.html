Module 1 — Core Feature Specifications

  • Add Product — Owner adds directly (immediate). Manager submits (goes to Pending Approval queue). Full field form with barcode scan-to-fill, camera photo capture, UoM conversion builder, location picker (drill-down tree).
  • Edit Product — Update any field on any existing product. Owner edits take effect immediately. Manager edits are queued. Complete edit history maintained with before/after values in the audit log.
  • Deactivate / Discontinue — Products are never hard-deleted. Owner can deactivate (hides from search, preserves all movement history) or mark as Discontinued. Both states are reversible.
  • Bulk Import from Excel — Owner uploads Excel file. System runs 28 validation rules (duplicate barcodes, missing required fields, invalid category paths, UoM mismatch, etc.). Returns a validation report with error row numbers. Clean rows imported in batches of 1,000. Full 26,000 SKU load completes within 3 minutes via background queue.
  • Barcode Scan Search — Any user opens the scanner screen, points the camera at any barcode or QR code, and instantly sees the product record with current stock level, location, and photo. Response under 300ms online, under 150ms offline (WatermelonDB).
  • Full-Text Search — Type-ahead search across product name, SKU code, barcode, model number, brand, and tags. MeiliSearch powers sub-800ms results across 26K SKUs with typo tolerance ("hafle handl blk" finds "Hafele Handle Black").
  • Category & Brand Browse — Drill down through the category tree (4 levels). Tap any brand to see all products for that brand with stock status badges. Filter any view by floor, section, or stock status.
  • Location Browse — Navigate by physical location: Floor → Section → Aisle → view all products at that exact location. Godown staff use this during cycle counts.
  • QR Label Generation — For products without manufacturer barcodes, the system generates a printable QR code label (contains SKU code, product name, location). Supports individual and batch label generation for unlabelled items.

Six Stock Status Flags — Automatic Intelligence

✅ In Stock — Adequate
Quantity is above the configured reorder point. No action needed. Green badge on product card and dashboard.
❌ Out of Stock — Zero Units
Quantity is 0. Immediate push notification to Owner. Appears at the top of the Out-of-Stock list on the dashboard.
⚠️ Low Stock — Below Reorder Point
Quantity has dropped below the Owner-configured reorder point. Amber alert on dashboard. Owner marks "Reorder Initiated" to silence.
💤 Dead Stock — No Movement 90+ Days
Stock exists but zero movements (inward or outward) for 90 days (configurable). Owner sees total ₹ value locked in dead stock.
📉 Out of Trend — Owner Flagged
Owner manually marks a SKU as out of market trend. Shown separately for clearance planning. New stock should not be ordered.
🐌 Slow Moving — Low Turnover
Selling, but slower than category peers. System flags for Owner review — price adjustment, promotion, or investigation.

🔐
The Maker-Checker Rule — Core Security Principle
Every stock quantity change, new product submission, or movement record initiated by a Sales Manager, Floor Staff, or Godown Staff is held in a "Pending Approval" queue. It does NOT affect live inventory levels until the Owner explicitly reviews it and approves. Owner-initiated changes take effect immediately. This ensures the Owner has seen and validated every piece of inventory data in the system from Day 1.

A single Laravel 12 backend API serves both the React Native mobile app and the Vue.js web panel. B6 · Security
Security Requirements
RequirementImplementation
Transport EncryptionTLS 1.3 on all API calls. Cloudflare handles SSL. HTTP strictly redirects to HTTPS.
AuthenticationPhone number + 6-digit PIN. JWT via Laravel Sanctum. Biometric re-auth (fingerprint/Face ID) within active session.
Token ExpiryOwner: 24 hours. All other roles: 10 hours. Refresh tokens: 30 days. Auto-refresh in background while session is active.
Device BindingMax 2 registered devices per user account. Fingerprint = Device ID + app install UUID (SHA-256). New device requires Owner approval.
RBAC EnforcementEvery API endpoint checks role via Laravel Policies at the controller level. Staff cannot call owner endpoints via Postman or any tool.
Inventory IntegrityStock level updates wrapped in PostgreSQL transactions. All-or-nothing — no partial movement records possible.
Immutable Audit TrailAll actions written to activity_log. Table has no UPDATE/DELETE grants for any application database user.
Brute Force5 failed PINs → 30-minute lockout. Owner push notification sent on lockout. 200 API requests/token/minute rate limit via Redis.
Root DetectionApp refuses to run on rooted Android (RootBeer) or jailbroken iOS (DTTJailbreakDetection).
Auto-LockApp locks after 5 minutes of inactivity. PIN or biometric required to unlock.
