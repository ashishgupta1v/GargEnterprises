DESIGN STYLE DEFINITION — NEUMORPHISM:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Neumorphism (soft UI) uses ONE background color for all surfaces.
Elements appear to extrude FROM or be pressed INTO the background
using dual shadows — one light (top-left) and one dark (bottom-right).

Rules:
- ALL elements share the same base background: #EEF0F5
- Raised elements: box-shadow: 6px 6px 16px #D1D4DE, -6px -6px 16px #FFFFFF
- Pressed/Active: box-shadow: inset 4px 4px 10px #D1D4DE, inset -4px -4px 10px #FFFFFF
- NO hard borders. NO background colors different from base.
- Gradients used ONLY for: primary CTA buttons, status badges, icon fills, brand highlights
- Small shadow for small elements. Large shadow for large cards. Proportional.

DESIGN GUARDRAILS — WHAT NOT TO DO:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ❌ No dark mode. No dark backgrounds anywhere except toasts.
  ❌ No pure white (#FFFFFF) backgrounds — only #EEF0F5 as base.
  ❌ No hard borders (1pt solid lines on cards).
  ❌ No flat colored card backgrounds (no blue card, no green card).
  ❌ No heavy drop shadows (always balanced light + dark twin shadow).
  ❌ No gradients on backgrounds or card fills.
  ❌ No mixed border radii on the same card.
  ❌ No colored sidebars or headers.
  ❌ No emoji as functional icons.
  ❌ No purple gradients — use blue primary only.
  ❌ No white (#FFFFFF) card background on #EEF0F5 page (no visible depth).
  ❌ No more than 2 gradient elements visible on screen at once.
  ❌ No sharp corners (minimum 6pt radius everywhere).
  ❌ No text smaller than 10pt.
  ❌ No clashing gradient colors side by side (harmonious palette only).

Neumorphism — Core Principles

The design paradigm explained with live examples

Raised (Protruding)
Resting card element
shadow: 6px 6px 16px #D1D4DE,
-6px -6px 16px #FFFFFF
Inset (Pressed In)
Input or active state
shadow: inset 4px 4px 10px #D1D4DE,
inset -4px -4px 10px #FFFFFF
Gradient Accent
CTA button or badge
gradient + colored shadow
for emphasis only
Key Rule: The background (#EEF0F5) is the same for ALL elements. Depth comes entirely from shadows, never from color contrast between surface and background. This creates the "extruded from one piece" feeling.

Accessibility & Design Guardrails

Contrast requirements and what never to do

Contrast Ratios
#1A202C on #EEF0F511.2:1 ✓
White on #4F7BF74.9:1 ✓
#718096 on #EEF0F54.7:1 ✓
Minimum body text4.5:1 WCAG AA
Design Guardrails
✅ Always use #EEF0F5 as page background
✅ Paired twin shadows (light + dark always together)
✅ Min touch target 44pt × 44pt
✅ Status shown by icon + color (not color alone)
✅ Min text size 11pt everywhere
❌ Never use white (#FFF) as card background
❌ Never use only one-sided shadow
❌ Never gradient on card backgrounds
❌ Never mix shadow sizes on same card
❌ Never dark mode — light only
