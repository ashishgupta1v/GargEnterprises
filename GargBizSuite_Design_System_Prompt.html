APP NAME: GargBizSuite — Inventory & Catalog Management
CLIENT: Garg Enterprises, Ludhiana (Hardware & Modular Kitchen Dealer)
PLATFORM: Mobile App (iOS + Android, React Native)
PHASE: Phase 1 — Catalog Management + Inventory Management
USERS: 4 roles — Owner, Sales Manager, Floor Staff, Godown Operator
TONE: Enterprise SaaS, Professional, Trustworthy, Clean, Efficient

DESIGN PHILOSOPHY:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Design a clean, minimal, light-mode enterprise mobile application.
The app should feel like a premium B2B SaaS product — similar to 
Notion, Linear, or Stripe Dashboard — adapted for mobile.

Core principles:
- Clarity over decoration: every element serves a function
- Data-dense but never cluttered: generous white space
- Trustworthy and calm: no loud colors, no excessive gradients
- One primary action per screen: clear visual hierarchy
- Fast and scannable: users need information at a glance

  Sizes:       16pt (inline/caption), 20pt (standard UI), 24pt (nav bar), 32pt (feature icons)
  Color rule:  Icons always inherit parent text color. Never use solid icon fills.
  Nav icons:   20pt, active = Primary Blue filled, inactive = Muted grey outline

  Prohibited: Do NOT use colored icon backgrounds in lists/rows (only in empty states).

  ❌ No colored icon containers on list rows
  ❌ No border-radius > 12pt on cards (except bottom sheets = 20pt)
  ❌ No drop shadows on navigation bars

Brand Identity & Philosophy

The design soul of GargBizSuite

Design Philosophy
Clarity First: Every pixel has purpose. If it doesn't help the user, remove it.

Screen-by-Screen UX Specifications

Detailed annotation for each of the 10 core screens

▶ STITCH SCREEN SPEC — DASHBOARD
SCREEN: Dashboard
ROLE: Owner (full view)
TOP BAR: 
  - Left: User avatar (initials circle, 36pt, blue tint)
  - Center: "GargBizSuite" wordmark 17pt/Syne/700
  - Right: Bell icon (notification dot if alerts exist)
HERO GREETING: 
  - "Good Morning, Pankaj" — 22pt/Syne/800/#0D1117
  - Date: "Friday, 29 May 2026" — 13pt/mono/muted
  - 16pt vertical spacing
KPI SECTION:
  - Section header: "TODAY'S OVERVIEW"
  - 2×2 grid of KPI cards (full-width, equal)
  - Cards: Total SKUs / In-Stock / Alerts (red border) / Inventory Value
  - Each card: icon top-left, large number, label, trend line
MOVEMENTS SECTION:
  - Section header "RECENT MOVEMENTS" + "See All" link
  - List of 4 rows, each 72pt height
  - Shows: move-type badge, product name, qty (colored ±), time, status badge
ALERTS SECTION:
  - Section header "STOCK ALERTS" + badge count
  - Compact list: severity icon + product name + qty vs reorder
  - "View All 312 Alerts" button (secondary, full-width)
BOTTOM NAV: 5-tab nav bar
▶ STITCH SCREEN SPEC — CATALOG
SCREEN: Product Catalog
TOP BAR:
  - Back chevron (if not tab root) OR title "Catalog"
  - Right: Grid/List toggle icon
SEARCH BAR (below top bar, full-width, 44pt):
  - Pill shape (border-radius 100pt)
  - Background #F1F5F9 (grey-wash)
  - MagnifyingGlass icon left
  - Placeholder: "Search products, SKU, brand…"
FILTER CHIPS (horizontal scroll row, 40pt height):
  - "All" (active = blue filled pill), "Hafele", "Blum", "Hettich", 
    "Yale", "Godrej", "Bosch"
  - Inactive: border pill. Active: #EBF5FF bg, #1A56DB text, #BFDBFE border
RESULTS COUNT: "26,412 products" — 12pt/muted, left-aligned
PRODUCT LIST (standard list, rows):
  - Each row 72pt height
  - Left: Brand tag (colored pill 10pt) + Product name (14pt/600) + SKU (11pt/mono/muted) + stock bar (4pt, colored)
  - Right: Price (14pt/600/primary blue) + Status badge
  - Divider between rows: 1pt #F1F5F9 (inset 16pt left)
  - Tappable: navigates to Product Detail screen
EMPTY STATE: MagnifyingGlass illustration + "No products match your search" + "Clear Filters" button
▶ STITCH SCREEN SPEC — RECORD MOVEMENT
SCREEN: Record Movement
ENTRY POINT: FAB (center bottom nav) → Bottom Sheet → Select type
OR: Direct navigation from Dashboard
TOP BAR: "Record Movement" title + X close button
SEGMENTED CONTROL (4 options, 48pt height):
  - INWARD | OUTWARD | TRANSFER | ADJUST
  - Active segment: White bg + blue text + subtle shadow
  - Inactive: Grey-wash bg, muted text
  - Full width, equal segments
CONTENT (scrollable form):
  SECTION 1 — Reference Details:
    - Card container: white bg, 12pt border-radius
    - Fields: Supplier / Reference No. / Date / Location (dropdown)
    - Fields in 2-column grid on each row where labels fit
  SECTION 2 — Items:
    - Section header "ITEMS RECEIVED" + Scan button (right, outlined)
    - Inline mini-table: SKU | Product | Qty | Condition
    - Each row editable inline
    - "+ Add Item" row at bottom (tappable)
  SECTION 3 — Notes:
    - Single multiline textarea "Add Remarks…"
STICKY BOTTOM BAR (above keyboard):
  - "Submit for Approval" — Primary button, full-width
  - Small muted text: "This request will be sent to owner for review"
KEYBOARD BEHAVIOR: Form scrolls up, sticky bar stays above keyboard

Mobile-First UX Patterns

Thumb-zone awareness, gesture patterns, and mobile-specific behaviors

Thumb Zone Rules
🟢 Bottom 40% — Primary actions, FAB, bottom nav
🟡 Middle 40% — Content, list items, form fields
🔴 Top 20% — Navigation bar only, minimal interaction
All CTAs positioned in thumb-reachable zones. Search bar in upper-middle (reachable on stretch).
Minimum Touch Targets
📐 44pt × 44pt minimum for all interactive elements
📐 48pt height for all input fields and buttons
📐 56pt height for standard list rows
📐 64pt height for bottom navigation bar
📐 8pt gap minimum between adjacent touch targets
